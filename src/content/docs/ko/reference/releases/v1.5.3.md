릴리스 날짜: **2023-06-04** — [GitHub 릴리스](https://github.com/zeromicro/go-zero/releases/tag/v1.5.3)

## 변경 사항
* 활성화 prometheus 전역 switch 때 user 활성화 devsever 메트릭 통해 @zcong1993 에서 https://github.com/zeromicro/go-zero/pull/3169
* 지원 optional otel 전역 initialization 위한 #3284 통해 @JasonMing 에서 https://github.com/zeromicro/go-zero/pull/3292
* 추적 exporter, 추가 새로운 타입 `file` 통해 @Kangkeizai 에서 https://github.com/zeromicro/go-zero/pull/3298
* change 포트 로 6060 통해 기본값 에서 devserver 통해 @kevwan 에서 https://github.com/zeromicro/go-zero/pull/3191
* 제거 기본값 keepalive params 위한 NewClientWithTarget 통해 @zcong1993 에서 https://github.com/zeromicro/go-zero/pull/3208
* sqlc 추가 Set캐시WithExpire 메서드 통해 @lhcGinv 에서 https://github.com/zeromicro/go-zero/pull/3249
* expand retry 메서드 로 지원 타임아웃과 interval 제어 통해 @jiang4869 에서 https://github.com/zeromicro/go-zero/pull/3283
* 추가 메서드 label 위한 prometheus 미들웨어 메트릭 통해 @SleeplessBot 에서 https://github.com/zeromicro/go-zero/pull/3226
* 추가 RunSafe 사용하여 컨텍스트 통해 @runtu666 에서 https://github.com/zeromicro/go-zero/pull/3224
* 추가 otlptracegrpc otlptracehttp 헤더 지원 위한 Uptrace 통해 @toby1991 에서 https://github.com/zeromicro/go-zero/pull/3219
* 추가 OtlpHttpPath 설정 지원 위한 ZincObserve Telemetry 통해 @toby1991 에서 https://github.com/zeromicro/go-zero/pull/3271
* more minor improvements과 버그 수정
